Proposed
1
The active pill's label can actually animate. Today maxWidth is undefined when the pill is active (SessionStrip.tsx:785-787) — there is no numeric value pair, so width snaps to intrinsic. Option B wraps the label in a display:grid box and animates grid-template-columns: 0fr → 1fr, which interpolates to the intrinsic width. That keeps the active pill's uncapped min-w-0 flex-shrink + ellipsis behavior intact. Option A caps it at 320px, which is simpler but truncates long names earlier than today.
2
Stop disabling the transition on the pill that was just clicked. transition: pack.expanded.has(s.id) ? 'none' : … (SessionStrip.tsx:789) — and packSessions puts the active pill in expanded unconditionally (pack-sessions.ts:53), so the clicked pill is exactly the one with animation switched off. The 'none' still matters for suppressing repack churn, so it becomes conditional on why the pill expanded: animate on an active-id change, stay instant when only the packing solver changed.
3
Bubbles pop in, staggered, on-screen only. 260ms cubic-bezier(0.34, 1.56, 0.64, 1) — the same overshoot curve already used for the pills — from scale(.9) translateY(10px), with a 30ms × index stagger. Applied to .timeline-entry.in-view, so off-screen entries render already-settled and cost nothing. Outgoing session gets a faster 120ms ease-in shrink-away so the swap doesn't feel laggy.
4
Gated on reducedEffects and prefers-reduced-motion. The session strip currently has no reduced-motion gate at all. Toggle reducedEffects in the topbar to confirm both animations fall back to instant.
